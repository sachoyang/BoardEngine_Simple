# 검증 체크리스트 — 오목 · 체스 (+ 엔진 개선분)

> 이 세션에서 추가/변경한 것들을 **VS2022 에서 직접 빌드·실행하며 확인**하기 위한 목록입니다.
> 이 저장소가 있던 환경엔 MSBuild/VS 가 없어 컴파일·런타임 검증을 하지 못했습니다.
> 아래 항목을 위에서부터 순서대로 확인하세요. 기대 결과와 다르면 그 항목을 기록해 주세요.
>
> **변경 범위 요약**
> - 엔진 개선 2건: `RestartGame()` API, `Label` 컴포넌트 + `SetObjectText()` API
> - 게임 2종: 오목(기본 `GameLogic.cpp`), 체스(`Examples/Chess_GameLogic.cpp`)
> - 관련 파일: `Engine/Engine/{IEngineAPI.h, Engine.h, Engine.cpp, Label.h, Label.cpp, Engine.vcxproj}`,
>   `BoardEngine_SDK/include/*`, `MakeSDK.ps1`, `Examples/*`

## 0. 빌드 검증 (가장 먼저)

- [ ] `BOARD/Engine/Engine.sln` 을 VS2022 로 연다. `Engine.h` 최상단이 `#define STANDALONE_MODE 0` 인지 확인.
- [ ] 구성 **Debug | x64** 로 **솔루션 빌드**. 에러 0 개.
  - [ ] `Label.cpp` 가 컴파일 목록에 포함됨(Engine.vcxproj 에 등록됨).
  - [ ] `IEngineAPI` 순수 가상 3개 신규(`RestartGame`, `SetObjectText`)가 `Engine` 에서 모두 구현되어 **추상 클래스 인스턴스화 에러가 없음**.
- [ ] `GameLogic.dll` 이 `Engine/x64/Debug/` 에 생성됨. LNK1104(파일 잠금) 없음.
- [ ] ⚠ 기존 저장소의 prebuilt `Engine.exe`/DLL 은 **stale** 이다. 반드시 위처럼 재빌드한 바이너리로 실행할 것.
- [ ] (ABI) Engine 과 GameLogic 이 **같은 구성/런타임(/MDd)** 으로 함께 빌드됨.

> **왜 함께 재빌드해야 하나:** `IEngineAPI` 에 가상 함수를 append 하면 vtable 레이아웃이 바뀐다.
> Engine 만 새로 빌드하고 옛 DLL 을 쓰거나 그 반대이면 잘못된 슬롯을 호출해 크래시난다.

## 1. 엔진 개선 검증

### 1-A. `SetObjectText` / Label (오브젝트 위 글자)

체스를 로드하면 자연히 확인되지만, 핵심 포인트:

- [ ] 말 위에 글자(P·N·B·R·Q·K)가 **칸 중심에 정렬**되어 표시된다.
- [ ] **밝은 칸(백)·어두운 칸(흑) 모두에서 글자가 읽힌다** (대비 아웃라인 자동).
- [ ] **마우스 휠 줌 / 우클릭 드래그 패닝** 시 글자가 말과 함께 이동·확대된다(어긋나지 않음).
- [ ] STANDALONE_MODE 1 배포 빌드에서도 글자가 보인다(ImGui 프레임은 항상 렌더).
- [ ] 빈 문자열로 `SetObjectText(obj,"")` 하면 글자가 사라진다.

### 1-B. `RestartGame` (재대국 / 씬 리로드)

- [ ] 게임 종료 후 보드 클릭 → 씬이 **처음 상태로 복원**되고 첫 턴부터 다시 시작(크래시 없음).
- [ ] 착색됐던 돌/말이 원래대로 돌아온다(colorTint 원복).
- [ ] 에디터에서 **GameLogic ▸ Reload Now** 로도 `OnLoad` 가 재호출되어 초기화된다.
- [ ] (경계) `RestartGame()` 은 프레임 끝에서 처리되므로 클릭 핸들러가 그 프레임에 크래시하지 않는다.

## 2. 오목 검증 (기본 `GameLogic.cpp`)

**준비:** Board Generator → Rows 15 / Cols 15 / Tile Size 40 → Generate → File ▸ Save Scene →
GameLogic ▸ Load GameLogic.dll → **[ Play ]**.

- [ ] 시작 시 상단에 `Black's Turn` 표시.
- [ ] 타일 클릭 → 흑돌(거의 검정) 착점, 다음 클릭 → 백돌(거의 흰색). 번갈아 진행.
- [ ] `assets/click.wav` 효과음 재생.
- [ ] 이미 놓인 칸 클릭 → `Already occupied!` 표시, 상태 변화 없음.
- [ ] **좌표 매핑 확인(중요):** 맨 왼쪽 위 / 맨 오른쪽 아래 **모서리 타일**을 클릭해도 정확히 그 칸에 착점된다.
  - ⚠ 이 게임은 타일이 **월드 원점(0,0)에서 시작 · Tile Size 40** 이라고 가정한다(`col = x/40`).
    보드를 다른 위치/크기로 만들면 매핑이 어긋난다 → 그 경우도 한 번 확인해 한계로 기록.
- [ ] **승리 판정:** 가로 5 / 세로 5 / ↘대각 5 / ↙대각 5 를 각각 만들어 `Black Wins!` 또는 `White Wins!  (click to rematch)` 표시.
  - [ ] 판 **가장자리에 걸친 5목**도 올바르게 인정(경계에서 방향 탐색이 break 로 끊기는지).
- [ ] **무승부:** (로직 확인) 225칸이 다 차고 승자 없으면 `Draw!  (click to rematch)`. 실제로 채우기 어렵다면 코드 리뷰로 대체.
- [ ] **재대국:** 승리/무승부 후 아무 곳 클릭 → 빈 판에서 흑부터 재시작.
- [ ] **핫 리로드:** `GameLogic.cpp` 를 조금 고치고 GameLogic 만 재빌드 → 0.5초 내 자동 반영.

## 3. 체스 검증 (`Examples/Chess_GameLogic.cpp`)

**준비:** `Examples/Chess_GameLogic.cpp` 내용을 `GameLogic/GameLogic.cpp` 에 덮어쓰기 →
Board Generator **Rows 8 / Cols 8 / Tile Size 64** → Save Scene → Load DLL → **[ Play ]**.

### 3-A. 배치·표시
- [ ] 32개 말이 표준 위치에 생성. 백은 하단(밝은 칸+어두운 글자), 흑은 상단(어두운 칸+밝은 글자).
- [ ] 후열 글자 순서 `R N B Q K B N R`, 그 앞줄은 전부 `P`.
- [ ] 시작 시 `White's Turn`.

### 3-B. 선택
- [ ] 내 말 클릭 → 초록 하이라이트. 같은 말 다시 클릭 → 해제. 다른 내 말 클릭 → 선택 이동.
- [ ] 상대 턴에 상대 말 클릭은 선택되지 않음.

### 3-C. 이동 규칙 (각 말 최소 1회)
- [ ] **폰:** 첫 수 1칸/2칸 전진, 이후 1칸. 정면 막히면 불가. **대각선은 적 있을 때만** 캡처.
- [ ] **나이트:** L자 이동, 다른 말을 넘어감.
- [ ] **비숍/룩/퀸:** 경로에 말이 있으면 막힘(통과 불가). 빈 길로만 이동.
- [ ] **킹:** 한 칸.
- [ ] 아군이 있는 칸으로는 이동 불가. 규칙 위반 클릭 → `Invalid move`.

### 3-D. 체크 / 체크메이트 / 스테일메이트
- [ ] 자기 킹이 잡히게 되는 수는 거부 → `Illegal - your king is in check`.
- [ ] 상대를 체크하면 다음 턴 안내에 ` - Check!` 접미사 표시.
- [ ] **체크메이트 재현(바보 메이트, 4수):** 아래 순서대로 (각 항목 = 출발칸 클릭 → 도착칸 클릭). 좌표는 `(row, col)`, row 0=상단, col 0=a열.
  1. 백 f폰: `(6,5) → (5,5)`
  2. 흑 e폰: `(1,4) → (3,4)`
  3. 백 g폰: `(6,6) → (4,6)`
  4. 흑 퀸: `(0,3) → (4,7)`  ← 여기서 `Black Wins - Checkmate!  (click to rematch)` 가 떠야 함.
- [ ] (선택) 스테일메이트 상황을 만들면 `Stalemate - Draw!` 표시(재현이 번거로우면 코드 리뷰로 대체).

### 3-E. 기타
- [ ] **폰 승급:** 폰이 끝 행 도달 → 글자가 `Q` 로 바뀌고 퀸처럼 움직인다.
- [ ] **캡처:** 적 말을 잡으면 그 말이 사라진다(다음 프레임). 킹은 합법 수 필터 때문에 **실제로 잡히지 않는다**(대신 체크메이트로 종료).
- [ ] 종료 후 클릭 → 재대국.

## 4. 의도적으로 넣지 않은 것 (한계 — 버그 아님)

확인해서 "없음"이 정상임을 기록:

- 오목: 렌주(금수) 룰 없음. 보드 원점·타일크기 하드코딩 가정(위 2 참고).
- 체스: **캐슬링 / 앙파상 / 언더프로모션 없음**, 승급은 항상 퀸 고정, 무한 반복·50수 무승부 미구현.
- 엔진: 오브젝트 위 글자는 ImGui 기본 폰트라 **ASCII 위주**(한글/체스 유니코드 기물 글리프는 표시 안 됨 → 그래서 알파벳 사용).
